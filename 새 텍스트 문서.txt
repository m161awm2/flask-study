좋아. 네가 하려는 건 보통 이렇게 가면 돼.

순서부터 말하면

VPC 만들기
서브넷 만들기
퍼블릭 서브넷 1개 이상
프라이빗 서브넷 2개 이상
인터넷 게이트웨이(IGW) 연결
라우팅 테이블 설정
EC2는 퍼블릭 서브넷에 생성
RDS는 프라이빗 서브넷에 생성
보안 그룹 연결

제일 처음엔 VPC부터 만드는 게 맞아.

네 상황에서 가장 쉬운 구조

처음이면 이렇게 해.

VPC 1개
퍼블릭 서브넷 1개 → EC2 넣기
프라이빗 서브넷 2개 → RDS 넣기
IGW 연결
퍼블릭 라우팅 테이블 연결
EC2 보안 그룹: 22, 80 정도 허용
RDS 보안 그룹: 3306을 EC2 보안 그룹에서만 허용

왜 프라이빗 서브넷 2개냐면
RDS는 보통 DB Subnet Group 때문에 서로 다른 AZ의 서브넷 2개 이상을 요구하는 경우가 많아.

1. VPC 만들기

AWS에서

VPC → Create VPC

설정 예시:

Name: my-vpc
IPv4 CIDR: 10.0.0.0/16

이렇게 만들면 돼.

10.0.0.0/16은 네 전용 큰 네트워크 범위라고 보면 됨.

2. 서브넷 만들기

VPC 만든 뒤 서브넷 생성.

예시:

public-subnet-a → 10.0.1.0/24
private-subnet-a → 10.0.2.0/24
private-subnet-c → 10.0.3.0/24

가능하면 AZ도 나눠:

ap-southeast-2a
ap-southeast-2c

이런 식으로.

구성 느낌:

10.0.1.0/24 → EC2용
10.0.2.0/24, 10.0.3.0/24 → RDS용
3. 인터넷 게이트웨이 연결

Internet Gateway 생성하고
방금 만든 VPC에 Attach 해.

이걸 해야 퍼블릭 서브넷의 EC2가 인터넷이 됨.

4. 라우팅 테이블 설정
퍼블릭 라우팅 테이블

새 라우팅 테이블 만들고 VPC 선택 후:

라우트 추가:

0.0.0.0/0 → Internet Gateway

그 다음 이 라우팅 테이블을

public-subnet-a
에 연결

이러면 그 서브넷은 퍼블릭처럼 동작함.

프라이빗 쪽

RDS만 넣을 거면 일단 프라이빗 서브넷은
인터넷 라우트 없이 둬도 됨.

5. EC2 생성

EC2 만들 때 네가 만든
VPC 선택
public-subnet-a 선택

그리고 중요한 거:

Auto-assign public IP 활성화

그래야 외부에서 SSH나 웹 접속 가능함.

보안 그룹 예시:

SSH 22 → 네 IP만 허용
HTTP 80 → 0.0.0.0/0
HTTPS 443 → 필요하면 0.0.0.0/0
6. RDS 생성

RDS 만들 때:

VPC: 네가 만든 VPC
DB subnet group: private-subnet-a + private-subnet-c
Public access: No

이렇게 하면 됨.

즉, RDS는 인터넷에 직접 안 보이게 두는 거야.

7. 보안 그룹 연결

이게 핵심임.

EC2 보안 그룹

예:

22 → 내 IP
80 → 전체
443 → 전체
RDS 보안 그룹

예:

MySQL/Aurora 3306
Source: EC2 보안 그룹

이게 중요함.
0.0.0.0/0로 열지 말고 EC2 보안 그룹만 허용해야 함.

즉:

인터넷 → EC2는 가능
EC2 → RDS는 가능
인터넷 → RDS는 불가

이 구조가 제일 기본적이고 안전해.

네가 진짜 먼저 할 것

헷갈리지 않게 딱 말하면:

1단계

VPC 생성 (10.0.0.0/16)

2단계

서브넷 3개 생성
public: 10.0.1.0/24
private: 10.0.2.0/24
private: 10.0.3.0/24

3단계

IGW 생성 후 VPC 연결

4단계

퍼블릭 라우팅 테이블 만들어서
0.0.0.0/0 -> IGW
설정하고 public subnet 연결

그 다음에

EC2 생성
RDS 생성
하면 돼.
제일 쉬운 팁

AWS에서 “VPC and more” 같은 자동 생성 마법사를 쓰면
VPC, 서브넷, 라우팅, NAT까지 한 번에 만들어주기도 함.

근데 너는 공부 중이니까
한 번은 수동으로 만들어보는 게 훨씬 좋다.
